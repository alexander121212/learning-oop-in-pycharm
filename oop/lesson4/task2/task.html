Encapsulation - restrict access to the objects of the components ( methods and variables ) . Encapsulation makes some of the components are accessible only within the class .

Encapsulation in Python works only at the level of agreement among programmers about what attributes are public and which - internal.

Single underlining in the beginning of the name attribute indicates that a variable or method is not intended for use outside the class methods , however, is available on the attribute name.

class A:
     def _private(self):
         print("This is private method!")

a = A()
a._private()

The double underscore at the beginning of the name attribute gives more protection : the attribute is unavailable for this name.


class B:
     def __private(self):
         print("This is private method!")

b = B()
b._B__private()

However, it does not fully protect , as an attribute still remains available under the name _ClassName__AttributeName
